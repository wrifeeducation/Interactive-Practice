## Project Overview

**Name:** WriFe Interactive Practice
**Purpose:** A gamified learning platform that wraps 61 existing lesson HTML files in a game shell with XP, lives, streaks, badges, and a World Map. Pupils complete interactive writing and grammar activities; teachers monitor progress through a class dashboard.
**Target users:**
- **Pupils** aged 7–14 — play lessons, earn XP/badges/stars
- **Teachers** — view class dashboard (heatmap, pupil profiles, common mistakes), manage leaderboard toggle
- Post-MVP: school admin, parent

## Mandatory Conventions

Every agent MUST follow all of the rules below without exception.

### 1. CSS / Colours
- Use CSS custom properties (variables) for ALL colours in component files.
- **Never hardcode hex values** (e.g., `#667EEA`) directly in `.tsx` or `.css` component files.
- All design tokens are defined in `src/index.css`. Reference them with `var(--token-name)`.

### 2. TypeScript Interfaces
- **All interfaces and types live in `src/types/index.ts` only.**
- Never create interfaces inline in component files or in separate per-file type files.
- Always read `src/types/index.ts` before implementing any feature to check for existing types.

### 3. Component Size
- Components must be **< 200 lines**. If a component exceeds this, split it into sub-components in a dedicated sub-folder.
- Prefer composition over monolithic components.

### 4. Accessibility & Test Hooks
- Add `data-testid="<descriptive-name>"` to **all interactive elements** (buttons, inputs, drag targets, etc.).
- Add `data-tts="<content-description>"` to **all text content elements** to support future text-to-speech integration.

### 5. Supabase Access
- All Supabase queries must go through `src/lib/supabase.ts`.
- Never import the Supabase client directly in components, pages, or stores.
- Always use Supabase RLS — never bypass with service role key on the client.

### 6. State Management
- **React Query** for all server state (fetching, caching, mutations).
- **Zustand** for UI state and auth state (lives, XP, streak, current user session).
- Do not use React context for state that belongs in Zustand or React Query.

### 7. TypeScript Strict Mode
- `tsconfig.json` has `"strict": true`. No `any` types are permitted.
- After every implementation session, run `npx tsc -b --noEmit` and fix all errors before reporting done.

### 8. Lazy Loading
- The Teacher Dashboard page and any future Admin pages **must** be lazy-loaded with `React.lazy()` and `<Suspense>`.
- Heavy components (e.g., confetti libraries) should also be lazy-loaded.

### 9. Vercel SPA Routing
- `vercel.json` must always contain the SPA rewrite rule so client-side routing works on direct URL access:
  ```json
  {
    "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
  }
  ```

### 10. Environment Variables
- `.env` must **never** be committed to Git. It is in `.gitignore`.
- All env vars are prefixed `VITE_` and accessed via `import.meta.env.VITE_*`.

### 11. Content Source
- The 61 lesson HTML source files live at `/sessions/ecstatic-trusting-darwin/mnt/WriFe Lessons/Lesson_01/` through `Lesson_61/`.
- `content/lessons.json` is generated by `scripts/parse-lessons.js` — do not hand-edit it.
- Activities are seeded into the Supabase `activities` table.

## Gamification Summary

- **XP:** 10 pts first correct attempt, 5 pts on retry; never decreases
- **Lives:** 5 per session; reaching 0 → Rest screen (soft block, not hard stop)
- **Stars:** 1–3 per lesson based on accuracy %, shown on World Map nodes
- **Streak:** Daily flame counter; milestone badges at 3/7/14/30/60 days
- **World Boss Challenge:** 15-question mixed review; awards 500 XP + World Badge; required to unlock next world
- **Badge counts:** 61 lesson badges + 6 world badges + 5 streak badges + 5 mastery badges + 3 speed badges = 80 total
